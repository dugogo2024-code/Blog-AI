# 블로그 포스트 자동화 시스템 PRD (Product Requirements Document)

## 1. 개요

키워드 입력만으로 관련 정보를 수집하고, 이를 바탕으로 블로그 포스트 초안을 작성하며, 최종적으로 Quarto를 사용하여 GitHub Pages에 게시하는 전 과정을 자동화하는 시스템을 구축합니다. 이 시스템은 콘텐츠 제작 프로세스의 효율성을 극대화하는 것을 목표로 합니다.

## 2. 목표

- **시간 절약:** 정보 검색, 초안 작성, 게시 등 반복적인 작업을 자동화하여 콘텐츠 제작에 소요되는 시간을 대폭 단축합니다.
- **콘텐츠 품질 향상:** 최신 정보를 신속하게 수집하고, 생성형 AI를 활용하여 콘텐츠의 질을 높입니다.
- **효율적인 워크플로우:** 개발자와 콘텐츠 제작자가 최소한의 개입으로 블로그 포스트를 발행할 수 있는 간결하고 효율적인 워크플로우를 제공합니다.

## 3. 사용자

- **블로거 및 콘텐츠 제작자:** 기술 블로그, 뉴스, 리뷰 등 다양한 주제의 콘텐츠를 정기적으로 발행하는 개인 또는 팀
- **개발자:** 문서 작성 및 공유 과정을 자동화하고자 하는 개발자

## 4. 기능 명세

### 4.1. 키워드로 정보 수집

- **입력:** 사용자가 제공하는 특정 키워드
- **프로세스:**
    1.  Google 검색 API를 활용하여 키워드와 관련된 최신 웹 문서, 뉴스, 기사 등을 수집합니다.
    2.  수집된 정보의 핵심 내용을 요약하고 정리합니다.
- **출력:**
    - `search.md`: 수집 및 요약된 정보가 저장된 마크다운 파일

### 4.2. 이미지 생성

- **입력:** 블로그 포스트의 주제 또는 핵심 키워드
- **프로세스:**
    1.  Gemini 이미지 생성 API에 키워드를 전달하여 블로그 포스트의 대표 이미지 생성을 요청합니다.
    2.  생성된 이미지를 지정된 형식과 크기로 다운로드합니다.
- **출력:**
    - `image.jpg`: 블로그 포스트에 사용될 이미지 파일

### 4.3. 블로그 포스트 초안 작성

- **입력:**
    - `search.md` (수집된 정보)
    - `image.jpg` (생성된 이미지)
- **프로세스:**
    1.  `search.md`의 내용을 기반으로 서론, 본론, 결론 구조를 갖춘 블로그 포스트 초안을 생성합니다.
    2.  생성된 `image.jpg`를 포스트의 적절한 위치(예: 상단)에 삽입하는 마크다운 구문을 포함합니다.
    3.  Quarto 형식에 맞는 메타데이터(제목, 작성자, 날짜 등)를 자동으로 추가합니다.
- **출력:**
    - `draft.md`: 완전한 형태의 블로그 포스트 마크다운 초안

### 4.4. Quarto를 사용한 게시 자동화

- **입력:** `draft.md`
- **프로세스:**
    1.  지정된 디렉토리(예: `posts/YYYY-MM-DD-post-title/`)를 생성합니다.
    2.  `draft.md` 파일의 이름을 `index.qmd`로 변경하고, `image.jpg`와 함께 해당 디렉토리로 이동시킵니다.
    3.  Quarto CLI를 실행하여 `index.qmd` 파일을 HTML로 렌더링합니다. (`quarto render`)
    4.  생성된 `_site` 디렉토리의 내용을 GitHub 리포지토리의 `gh-pages` 브랜치 또는 메인 브랜치의 `/docs` 폴더에 푸시하여 GitHub Pages에 게시합니다.
- **출력:**
    - `posts/YYYY-MM-DD-post-title/index.html`: 최종적으로 웹에 게시될 HTML 파일
    - 업데이트된 GitHub Pages 사이트

## 5. 기술 스택 (제안)

- **정보 수집:** Google Custom Search API
- **이미지 생성:** Google Gemini API (Image Generation)
- **초안 작성:** Google Gemini API (Text Generation)
- **게시:** Quarto, Git, GitHub Actions
- **스크립트:** Python 또는 Shell Script

## 6. 전체 작업 흐름

1.  **실행:** 사용자가 메인 스크립트를 실행하며 '키워드'를 인자로 전달합니다.
    - 예: `python run.py "제미나이 API 최신 업데이트"`
2.  **정보 수집:** 스크립트가 Google 검색으로 "제미나이 API 최신 업데이트" 관련 정보를 수집하여 `search.md`에 저장합니다.
3.  **이미지 생성:** 스크립트가 Gemini API로 관련 이미지를 생성하여 `image.jpg`로 저장합니다.
4.  **초안 작성:** 스크립트가 `search.md`와 `image.jpg`를 조합하여 `draft.md`를 생성합니다.
5.  **게시:**
    - `posts/2026-01-17-gemini-api-update`와 같은 디렉토리를 생성합니다.
    - `draft.md`는 `index.qmd`로, `image.jpg`는 그대로 새 디렉토리로 이동됩니다.
    - GitHub Actions 또는 로컬 스크립트가 `quarto render` 명령을 실행합니다.
    - 변경 사항이 GitHub 리포지토리에 푸시되고, 잠시 후 GitHub Pages에 새로운 글이 게시됩니다.

## 7. 산출물

- `PRD.md`: 본 문서
- `search.md`: 키워드 검색 결과
- `image.jpg`: 생성된 대표 이미지
- `draft.md`: 블로그 포스트 초안
- `posts/{post-name}/index.html`: 최종 게시물 HTML 파일
